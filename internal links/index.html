Here's an example of how to use anchor links on your pages. This is just straight html, no CSS. Above are some links to some paragraphs. You can hit the links above and see what the result is.

Step 1

The first thing you're going to do is to create your paragraphs (obviously). You need to have content on the page before linking to it. So create some paragraphs.

Step 2

The second thing you'll do is to give each of your anchors an ID attribute. (By anchors, I mean the item you want to pop to the top of the page. Here, I've anchored items in the <h3> elements, because I want the h3 to be the first thing displayed on the page when you hit the link.) The ID attribute is used because it's unique to an item. Class attributes would work as long as you had a unique name for each one, but that's a janky way of doing the code and will make people reading it unhappy. So create some IDs. Here, I've called them "first," "second," "third" and "fourth."

Step 3

Then, go ahead and create your list for navigation. I've just named mine first, second, third and fourth. (Actually, you can do this first, it doesn't really matter, but you might as well do it after you create the ID hooks so you can write the link code at the same time.)

Step 4

Then, to write the link code, just structue it like you see in the source up top. To be specific, write:

<a href="#first">Step 1</a>

<a href="#second">Step 2</a>

<a href="#third">Step 3</a>

<a href="#fourth">Step 4</a>

To link to the top of the page, you can use a blank # reference, like so:

<a href="#">Top of page</a>

That should be all you need to do. To test this, though, you're going to have to have some long fake text on the bottom of your pages. The browser won't go to the top of the paragraph if there's no more page below it to render. That's an awkward explanation, but I think you get the point. You can see this principle illusrated at this link; the bottom of the page won't wind up at the top of your browser window.
